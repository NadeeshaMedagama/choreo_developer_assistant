When adding new documentation:
1. Determine the appropriate category directory
2. Create the markdown file in that directory
3. Add a link to the directory's README.md
4. Update this INDEX.md if it's a major document
5. Follow the existing naming conventions

### Naming Conventions
- Use `SCREAMING_SNAKE_CASE.md` for compatibility
- Be descriptive: `FEATURE_NAME_IMPLEMENTATION.md`
- Use prefixes: `QUICK_START_`, `FIX_`, `TROUBLESHOOTING_`
